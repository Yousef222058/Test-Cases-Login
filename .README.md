# Software Testing Project: Login Page Test Cases

**Submission Date:** 16/04/2026
**GitHub Repository Name:** Test Cases – Login

## Project Overview

This document outlines the systematic testing approach applied to a standard Login Page. The testing covers two main categories: Functional requirements including basic security, and Non-functional requirements focusing on input validation using Boundary Value Analysis.

---

## Task 1: Functional and Security Test Cases

**Objective:** Verify that the login page authenticates users correctly, handles errors appropriately, and is secure against basic SQL injection attacks.

| Test Case ID | Description | Steps to Execute | Expected Result | Status |
| :--- | :--- | :--- | :--- | :--- |
| **TC_01** | **Valid login** | 1. Open login page.<br>2. Enter a registered, valid username.<br>3. Enter the correct matching password.<br>4. Click 'login' button. | User successfully logs in and is redirected to the dashboard/home page. | Not Executed |
| **TC_02** | **Invalid login** | 1. Open login page.<br>2. Enter an incorrect username and/or password.<br>3. Click 'login' button. | Login fails. Clear error message displayed: **"Invalid credentials"**. | Not Executed |
| **TC_03** | **Empty fields** | 1. Open login page.<br>2. Leave username and password fields empty.<br>3. Click 'login' button. | Login fails. Validation message displayed: **"Fields cannot be empty"**. | Not Executed |
| **TC_04** | **SQL Injection (Basic)** | 1. Open login page.<br>2. Enter basic SQL injection payload `' OR '1'='1` in the username field.<br>3. Enter any character string as password.<br>4. Click 'login' button. | Login must fail. The system must prevent unauthorized access via SQL injection and display an appropriate error. | Not Executed |

## Task 2: Boundary Value Testing

**Objective:** Validate input restrictions on the Password field using Boundary Value Analysis (BVA), ensuring it adheres to the length requirement of **8 to 12 characters (inclusive)**.

| Test Case ID | Description | Input (Password Length) | Expected Result | Status |
| :--- | :--- | :--- | :--- | :--- |

| **TC_BV_03** | Maximum boundary | **12 characters** | System **ACCEPTS** input. Password validation successful (for this constraint). | Not Executed |
